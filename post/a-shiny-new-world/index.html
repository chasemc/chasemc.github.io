Introduction

I recently gave a talk at a meetup in Chicago about an R Shiny app I am working on called IDBac. While there are plenty of example apps for Shiny I also wanted to create some examples of my own for now and any future instruction. Since the meetup had some scheduled time for attendees to work in groups or on their own I thought a nice demonstration of Shiny would be an app that could interactively create groups based on a self-reported skill level.

However I also wanted to include some introductory-type apps to cover a range of beginner to intermediate skill levels. So, in all, there are three suggested apps to look through, hopefully with each stepping up a degree in skill level needed for both data-wrangling and Shiny skills. Some links to tutorials and resources can be found at the end of the post.

  • App One
    • Was borrowed from RStudio’s example gallery and should be straightforward enough to learn some of the basics of Shiny as you go through tutorials.
      • Skills: simple data analysis, simple clustering, simple plotting with base R
  • App Two
    • Is based on the GapMinder dataset and includes two static ggplots and an interactive datatable.
      • Skills: simple data analysis, simple plotting with ggplot
  • App Three
    • Is the demonstration app on splitting Meetup attendees into groups. It uses googlesheets, jsonlite to parse API responses, and some more advanced data analysis (both purrr and lapply) and building a dynamic nested HTML table in Shiny.
      • Skills: Parsing API and JSON, googlesheets,dplyr, purrr (map), lapply, dynamic html tags in Shiny There are some challenges/suggestions included in each of the apps.

App One: Beginner Example

I find Joe Cheng’s k-means example to be pretty straightforward to point out the basic features of reactivity and chose this as challenge for beginner’s to get familiar with shiny. App Three: Interactively Create Groups at a Meetup

Due to some technical difficulties with the display tech at the meetup I wasn’t able to present this app example since I was using a computer sans-R, but wanted to include it here.

Imagine:

You’re organizing an Rstats workshop through Meetup.com. You have a lot of participants and want to separate them into groups based on self-reported skill levels, how would you accomplish this with Shiny?

Of course there are many ways, but the simplest I thought of was to use Google Forms and connect responses to information retrieved from the Meetup.com API.

  1. First, you would need to setup a Google Form and send it to participants to fill out: Example Google Form here
    • The responses will be recorded into a Google Sheet.
      • For more information on using Google Sheets with R, please see Jenny Bryan and Joanna Zhao’s tutorial.
    • Copy the key for the Google Sheet (eg in bold below below)
      • https://docs.google.com/spreadsheets/d/ 1xxtw88dEKDdQ1CoeLm46FWmNt3bqPSCGp4P7LA7BNv0
    • Find the code snippet below within the shiny app and replace “Insert Key Here” with your key
responses <- reactive({
      surveyResponses <- gs_key(" Insert Key Here ")
      surveyResponses %>%  gs_read(ws = "Form Responses 1")
      
  })
  1. Then create the json API GET request for your Meetup.com event. To do so you would go here, fill out the form, create the API, and copy the Signed URL that is created.
    • Find the code snippet below within the Shiny app and replace “Insert Signed URL Here” with your signed url.
listFromAPI<-read_json(" Insert Signed URL Here ")
